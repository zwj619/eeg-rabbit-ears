

# Arduino与PCA9685扩展板：双舵机串口控制入门教程

通过 Arduino 开发板与 PCA9685 16路 PWM 驱动板，利用电脑串口发送指令（`1` 或 `0`），实现对连接在 0号 和 4号 通道上的两个舵机进行平滑且反向同步的运动控制。

### 第一步：准备材料

1. **Arduino 开发板**（如 Arduino Uno R3 或 Nano）
2. **PCA9685 16路 PWM 舵机驱动板**
3. **微型舵机 2 个**
4. **杜邦线** 若干

---

### 第二步：硬件接线指南

本教程采用简化接线方案，利用 Arduino 本身的 5V 输出同时为芯片逻辑和舵机动力供电。

#### 1. Arduino 连接 PCA9685 (控制与动力供电)

PCA9685 芯片采用 I2C 协议通信，请按以下方式连接引脚：

* **VCC** -> Arduino 的 **5V**（用于给驱动板上的控制芯片供电）
* **GND** -> Arduino 的 **GND**（共地）
* **SDA** -> Arduino 的 **A4** (Uno) 或对应的 SDA 引脚
* **SCL** -> Arduino 的 **A5** (Uno) 或对应的 SCL 引脚
* **V+** -> Arduino 的 **5V**（用于给舵机提供动力电源）



#### 2. 舵机连接 PCA9685

将两个舵机分别插入 PCA9685 的 **通道 0** 和 **通道 4** 的排针上。连接时需注意排针颜色与舵机线缆的对应关系：

* **信号线（通常为黄色或橙色）** -> 对应驱动板上的 **PWM（黄色排针）**
* **电源线（通常为红色）** -> 对应驱动板上的 **V+（红色排针）**
* **地线（通常为棕色或黑色）** -> 对应驱动板上的 **GND（黑色排针）**

### 第三步：安装依赖库

在编写和上传代码之前，需要安装支持 PCA9685 芯片的官方驱动库：

1. 打开 Arduino IDE。
2. 依次点击菜单栏：**项目 (Sketch)** -> **包含库 (Include Library)** -> **管理库 (Manage Libraries...)**。
3. 在弹出的搜索框中输入：`Adafruit PWM Servo Driver`。
4. 找到由 Adafruit 出品的同名库，点击 **安装 (Install)**。

### 第四步：核心控制逻辑设计

在开始编写具体代码之前，我们需要先构思好如何控制舵机。为了让动作更加智能和平稳，我们将要在代码中实现以下三个关键逻辑：

* **平滑控制（拆分步长）：**
如果直接向驱动板发送最终的目标角度，舵机会以最大速度猛烈甩过去或静止不动。为了实现平滑转动，我们可以利用 `for` 循环结构。核心思路是：将一次大角度的转动拆分成许多个极其微小的步长，并且在每走一小步之后加上短暂的延迟（例如 `delay(2)`），从而实现舵机缓慢、平滑的过渡转动。
* **多舵机同步联动（数学映射）：**
为了让两个舵机同时动作且方向相反，我们不需要分开写两次控制逻辑。我们可以在同一个 `for` 循环周期内同时向两个通道发送指令：让 0 号通道的数值随着循环不断递增，同时让 4 号通道的数值不断递减。这样就能利用数学关系，让两个舵机完美同步地进行反向运动。
* **状态记录与防重复防抖（引入状态锁）：**
在使用串口控制时，如果用户连续多次发送相同的指令（例如一直按 `1`），程序就会不断重复触发同一个动作，导致舵机卡顿或逻辑混乱。为了解决这个问题，我们需要在代码中引入一个“状态锁”变量（如 `lastValue`），用来记录系统上一次执行的指令。这样，只有当接收到的新指令与当前记录的状态不同时，系统才会放行并允许舵机执行新的动作。

### 第五步：完整程序代码

根据上述逻辑设计的完整 Arduino 程序如下。请将其复制到 Arduino IDE 中：

```cpp
/***************************************************
  基于 Adafruit 16-channel PWM & Servo driver 库的
  双舵机串口平滑联动控制程序
 ****************************************************/

#include <Wire.h>
#include <Adafruit_PWMServoDriver.h>

// 实例化驱动对象，使用默认 I2C 地址 0x40
Adafruit_PWMServoDriver pwm = Adafruit_PWMServoDriver();

// 根据舵机规格调整脉宽边界值（对应 0-4096 的计数）
#define SERVOMIN 150 // 舵机最小脉宽位置
#define SERVOMAX 400 // 舵机最大脉宽位置
#define SERVO_FREQ 50 // 模拟舵机工作频率，通常为 50 Hz

int value;          // 存储从串口读取到的当前字符
int lastValue = 0;  // 存储上一次执行的指令状态，用于状态锁

void setup()
{
    Serial.begin(115200); // 初始化串口通信，波特率设置为 115200

    pwm.begin();
    pwm.setOscillatorFrequency(27000000); // 设置板载振荡器频率
    pwm.setPWMFreq(SERVO_FREQ);           // 设置 PWM 频率

    delay(10);
    while (!Serial)
        ; // 等待串口监视器开启
    Serial.println("8 channel Servo test!");
}

void loop()
{
    // 检查串口缓存区是否有数据可读
    if (Serial.available() > 0)
    {
        value = Serial.read(); // 读取串口输入
        
        // 打印接收到的数据信息
        Serial.print("收到指令: ");
        Serial.write(value); 
        Serial.print(" (ASCII: ");
        Serial.print(value);
        Serial.println(")");

        // 当接收到指令 '1' 且当前状态不为 '1' 时执行
        if (value == '1' && lastValue != '1')
        {
            Serial.println("设置舵机到最小角度位置");
            
            // 使用 for 循环逐个步长递增，实现平滑移动
            for (uint16_t i = 0; SERVOMIN + i <= SERVOMAX; i++)
            {
                pwm.setPWM(0, 0, SERVOMIN + i); // 0号通道脉宽递增
                pwm.setPWM(4, 0, SERVOMAX - i); // 4号通道脉宽递减
                delay(2);                       // 步长间的微小延迟，控制转动速度
            }
            lastValue = value; // 更新状态锁
        }
        // 当接收到指令 '0' 且当前状态不为 '0' 时执行
        else if (value == '0' && lastValue != '0')
        {
            Serial.println("设置舵机到最大角度位置");
            
            // 使用 for 循环逐个步长递减，实现平滑移动
            for (uint16_t i = 0; SERVOMIN + i <= SERVOMAX; i++)
            {
                pwm.setPWM(0, 0, SERVOMAX - i); // 0号通道脉宽递减
                pwm.setPWM(4, 0, SERVOMIN + i); // 4号通道脉宽递增
                delay(2);                       // 步长间的微小延迟，控制转动速度
            }
            lastValue = value; // 更新状态锁
        }
        else
        {
            Serial.println("接收到其它指令: " + String(value) + " (ASCII: " + String(value) + ")");
        }
    }

    delay(1); // 进程微休眠，降低 CPU 占用率
}

```

### 第六步：运行与测试

1. 检查并确保所有硬件连线无误，尤其是供电线路切勿短路。
2. 使用 USB 线将 Arduino 连接至电脑，在 IDE 中选择正确的开发板和端口，点击 **上传 (Upload)**。
3. 上传成功后，点击 IDE 右上角的 **串口监视器 (Serial Monitor)** 图标。
4. 在串口监视器右下角，将波特率下拉菜单精确设置为 **115200 baud**。
5. 在上方的输入框中输入数字 `1` 并点击发送（或按回车）：
* 0号通道舵机将平滑移动至设定的最小位置，同时4号通道舵机平滑移动至最大位置。


6. 在输入框中输入数字 `0` 并点击发送：
* 两个舵机将同时向相反方向平滑返回原位。


7. 连续多次输入相同的数字（如连续发送多个 `1`），通过串口监视器可以观察到动作不会被重复触发。

### 第七步：故障排查指南

* **串口监视器显示乱码或无反应：**
检查串口监视器右下角的波特率是否已精确选择为 `115200`。若设置正确仍无输出，请检查 A4(SDA) 和 A5(SCL) 的通信线是否接反。
* **舵机发出间歇性震动、滋滋声，或导致电脑 USB 发出断开提示音：**
此现象代表**供电电流严重不足**。因为目前采用的是 Arduino 5V 直接供电，说明当前的舵机负载已经超出了主板的输出极限。此时**必须立即拔掉 `V+` 与 Arduino 5V 的连接**，改用独立的外部直流电源（如 5V/2A 电池组或适配器）接到驱动板绿色的接线柱上。
* **舵机在旋转到特定位置时卡住并发出剧烈机械摩擦声：**
由于不同品牌舵机的物理机械极限不同，代码中的 `SERVOMIN` (150) 和 `SERVOMAX` (400) 设定值可能超出了当前舵机的物理行程。需要根据实际表现，在代码顶部逐步调大 `SERVOMIN` 或调小 `SERVOMAX` 的数值。
